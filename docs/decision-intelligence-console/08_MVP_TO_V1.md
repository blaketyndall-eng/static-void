# MVP to V1

## What to build first
The first version should be brutally practical.

### MVP
- source/API registry
- opportunities table
- decision ledger
- shortlist/console application registry
- experiment tracker
- weekly recommendation digest
- simple evidence pack generator

### V1
- prediction engine
- suggestion engine
- opportunity scoring system
- app health monitoring
- agent management panel
- launch operating system
- richer evaluation workflows

## What not to overbuild first
- excessive autonomy
- too many agent roles
- fancy but low-signal dashboards
- unsupported prediction claims
- deep integrations without repeated use cases

## Rule
The MVP should already make the team noticeably smarter and faster.

V1 should make the system feel like a true internal operating advantage.

# API & External Intelligence Layer

## Goal
Create a robust map of free, no-auth, free-trial, and paid sources that improve the console and future apps.

## Strategic categories

### Economic / market context
- FRED
- BLS
- SEC EDGAR
- Census and related government datasets

### News / monitoring
- GNews or alternatives
- SerpAPI / search monitoring
- social or discussion APIs where useful

### Research / knowledge
- OpenAlex
- Crossref
- public filings and government records

### Company / vendor signals
- official docs
- review sources
- public changelogs
- pricing pages
- trust/security pages

### Operational / technical signals
- deployment logs
- uptime providers
- analytics APIs
- internal telemetry

## Console use cases
- opportunity detection
- vendor/evaluation support
- GTM intelligence
- macro and industry context
- automated watchlists
- source freshness scoring

## Design requirements
- every source gets metadata
- every source gets recommended use cases
- every source gets a trust/freshness view
- every recommendation should preserve provenance
- system should know when not to rely on a weak source

## Strategic benefit
A strong source layer becomes a compounding advantage across:
- Shortlist
- future niche applications
- internal research
- marketing and design intelligence
- prediction systems

# Build Roadmap

## Phase 0 — Foundation
- define entity model
- stand up source registry
- define core events and telemetry
- establish prompt/model versioning
- create internal design system for console

## Phase 1 — Console MVP
Ship:
- mission control
- opportunity radar
- source registry
- app registry
- experiment tracker
- recommendation digest

## Phase 2 — Shortlist reinforcement
Ship:
- decision workflow improvements
- evaluation evidence system
- dealbreaker/fine-print logic
- better compare layer
- collaborator workflow support

## Phase 3 — Prediction + suggestion systems
Ship:
- recommendation scoring
- launch/app health forecasting
- risk prediction
- next-best-action logic

## Phase 4 — Marketing/Design OS
Ship:
- messaging system
- launch planning system
- campaign experiment tracker
- asset workflow management

## Phase 5 — Multi-app engine
Ship:
- repeatable pattern for launching niche apps
- shared services across products
- common telemetry + experiment stack
- internal playbooks for app incubation

## Principle
Build the console as infrastructure that increases the quality and speed of every future move.

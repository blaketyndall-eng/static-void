# Agent System

## Design objective
Create a multi-agent system that improves outputs without turning into chaos.

## Agent families

### Research agents
- gather sources
- summarize findings
- identify contradictions
- flag missing evidence

### Structuring agents
- normalize inputs
- build schemas
- organize artifacts
- maintain entity consistency

### Scoring agents
- evaluate opportunities
- rank vendors/options
- estimate source trust
- score experiments

### Monitoring agents
- watch product/app health
- watch trends and markets
- detect anomalies
- trigger reviews

### Recommendation agents
- suggest next actions
- suggest tests
- suggest launches
- suggest system improvements

## Control principles
- explicit permissions
- auditable outputs
- versioned prompts/configurations
- clear escalation paths
- human review on high-impact outputs
- performance measurement over time

## Failure to avoid
- too many overlapping agents
- low-accountability recommendations
- unclear provenance
- overconfident predictions
- tool sprawl without measured value

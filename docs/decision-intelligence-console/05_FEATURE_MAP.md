# Feature Map

## Highest-priority console modules

### 1. Suggestion Engine
Purpose:
turn signals into recommended actions.

Outputs:
- build recommendations
- GTM recommendations
- pricing tests
- niche suggestions
- infra improvements
- anomaly triage

### 2. Prediction Engine
Purpose:
estimate what is likely to happen next.

Outputs:
- launch success probability
- experiment confidence
- incident risk
- opportunity quality scoring
- model/source trust adjustments

### 3. Opportunity & Trend Radar
Tracks:
- niches
- trends
- underserved markets
- emerging API/data openings
- competitor weakness

### 4. Evaluation Engine
Handles:
- vendor comparison
- build-vs-buy
- shortlist evaluation workflow
- evidence pack generation
- risk summaries

### 5. Source Registry
Tracks:
- source quality
- access type
- pricing
- freshness
- reliability
- where each source should be used

### 6. Experiment Tracker
Tracks:
- hypotheses
- tests in progress
- outcomes
- metrics
- learnings
- next actions

### 7. App Registry + Ops View
Tracks:
- all internal tools
- all external applications
- deployments
- incidents
- key metrics
- release readiness

### 8. Agent Workbench
Tracks:
- agent roles
- prompt versions
- tool permissions
- failure modes
- win/loss data
- improvement loops

## Design principle
Each module should either:
- sharpen decisions
- increase leverage
- reduce wasted motion
- improve shipped outcomes
